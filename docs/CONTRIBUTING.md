# Contributing to Dynamic Phase Coherence

Thank you for your interest in contributing to the **Dynamic Phase Coherence** repository!

We welcome independent research, code optimizations, and field synchronization models. To maintain the zero-friction integrity of this workspace, all contributions must adhere to the following guidelines.

---

## 1. Core Contribution Principles

* **No-Fluff Engineering:** Keep documentation, code comments, and commit messages direct, technical, and precise.
* **Dual Formula Formatting:** Any LaTeX equations introduced in Markdown documentation **must** be followed immediately by a plaintext/regular text equivalent for universal readability across all viewports.
* **Zero Ohmic Resistance:** Submissions should focus on reducing computational, systemic, or operational noise. High-entropy or speculative clutter will be rejected.

---

## 2. Pull Request Guidelines

1. **Fork the Repository:** Create your feature branch (`git checkout -b feature/phase-delay-optimization`).
2. **Test Your Code:** Ensure any Python scripts added to `/src` are fully executable and compatible with standard Python 3.x environments (including Pydroid 3).
3. **Document Context:** Update relevant files under `/docs` if your changes modify theoretical parameters or cycle formulas ($\Phi \to H \to C$).
4. **Commit Cleanly:** Use standard conventional commits (e.g., `feat: add phase anchor test`, `docs: clarify THz limit`).
